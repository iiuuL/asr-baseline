
基础的语音转文字和情感识别、语种鉴定及音频事件检测（如笑声、掌声

#### 0.环境部署

1. **前置系统要求** (必须)
在开始之前，请确保你的电脑已安装 FFmpeg（音频格式转换与切片解码的核心工具）

安装说明：下载并解压后，务必将 bin 文件夹路径添加到系统的 环境变量 (Path) 中。在 cmd 输入 ffmpeg -version 确认成功。

2. **搭建 Python 环境** (二选一)
方案 A：使用 Conda 一键还原 (推荐)
如果你电脑装了 Miniconda/Anaconda，直接运行：
```
conda env create -f environment.yml
conda activate asr-baseline
```
方案 B：使用 Pip 安装
如果你已有 Python 3.9+ 环境，请运行：
```
pip install -r requirements.txt
```

3. **获取模型权重** (避坑指南)
本项目使用 SenseVoice 模型，初次运行会自动下载约 3GB 的权重。
如果网络好：直接启动服务，程序会自动下载到 ~/.cache/modelscope。
如果下载慢：找我要离线权重包，并解压到对应目录。

4. **启动 ASR 服务**
在项目根目录下，进入虚拟环境，并且双击运行：
run_service.cmd
看到 Uvicorn running on http://127.0.0.1:18080 即代表成功。

5. **验证**
打开浏览器访问 http://127.0.0.1:18080/docs 确认接口文档可用。
双击打开 demo.html，允许麦克风权限，对着电脑说话。
如果看到“识别文本”和“情感状态”有跳动，说明联调成功。

#### 2. 切片上传逻辑

前端只需每秒发送当前的音频分片（Slice），后端会自动在内存中为每个 session_id 维护一个缓冲区。
增量识别：每次请求返回的 text 都是从当前对话开始到此刻为止的全量文本。
前端处理：前端展示时，应使用最新的返回结果覆盖旧结果，而不是在旧文字后面叠加。

所以随着**单次**会话时间的增加，后端需要处理的音频长度线性增长，识别耗时会增加。所以**建议单次 session 持续时间控制在 30 - 60 秒以内。**
>因为流式上传过于复杂，所以选了切片上传伪实时。如果30-60s可能不能完全满足需求，后续可能会优化

##### 结束上传

识别到一段完整的话后，通过 is_final: true 结束当前会话，并开启新的 session_id 以重置算力开销。
**必须发送 is_final**：当用户停止录音或准备切换话题时，务必发送一个 is_final: true 的请求。
作用：后端接收到该信号后会清理内存中的音频缓存。若不发送，会导致后端内存持续占用，甚至影响后续识别的准确性。
> 目前is_final的发送是根据录音停止，之后会优化成VAD识别单句
#### 3. 接口说明



 **语音识别（切片上传）**

* **方法**: `POST`
* **路径**: `/asr/chunk`
* **Content-Type**: `multipart/form-data`

 **请求参数**

| 字段名        | 类型    | 必填 | 说明                                                               |
| ------------- | ------- | ---- | ------------------------------------------------------------------ |
| `file`        | Binary  | 是   | 音频切片文件（支持 .wav, .mp3, .flac）。建议长度：500ms - 3000ms。 |
| `session_id`  | String  | 是   | 唯一会话 ID，用于追踪同一段对话。                                  |
| `chunk_index` | Integer | 是   | 切片索引，从 0 开始递增。                                          |
| `is_final`    | Boolean | 否   | 是否为最后一段切片。默认 `false`。                                 |
| `language`    | String  | 否   | 语种强制指定（如 `zh`），默认 `zh`。建议不传，让模型自动识别。     |

**返回数据结构 (JSON)**

```json
{
  "ok": true,
  "session_id": "text001",
  "chunk_index": 0,
  "is_final": true,
  "text": "我们来测试一下语音识别，这是一条中文短音频。",
  "raw_text": "<|zh|><|SAD|><|Speech|><|withitn|>我们来测试一下语音识别，这是一条中文短音频。",
  "metadata": {
    "language": "zh",
    "emotion": "SAD",
    "events": [
      "Speech"
    ],
    "has_speech": true,
    "itn_mode": "withitn",
    "unknown_tags": []
  },
  "language": "zh",
  "device": "cuda:0",
  "elapsed_ms": 530.781,
  "filename": "smoke.m4a"
}

```

#### 4. 情感与事件标签映射表

前端可根据 `metadata` 中的标签，预设数字人的表情动画。

 **情感标签**

| 标签        | 对应含义 |
| ----------- | -------- |
| `SAD`       | 悲伤     |
| `ANGRY`     | 愤怒     |
| `NEUTRAL`   | 中立     |
| `SURPRISED` | 惊讶     |

 **特殊事件**

* `Laughter`: 检测到笑声。可以驱动数字人同步发出笑声动画。
* `Applause`: 检测到掌声。

#### 5. 可能的问题 

**乱码**：接口所有字符串均使用 `UTF-8` 编码，请确保前端解析时编码一致。
**音频格式**：
- 采样率： 建议前端采集时设置采样率为 16000Hz，单声道（Mono）。
- 容器：demo.html 中使用的是浏览器默认编码，后端通过 FFmpeg 自动兼容。若前端使用自定义录音库，请确保格式为标准的 WebM、OGG 或 WAV。
